Challenge 7 — CSP: The Defence That Would Have Stopped It Intermediate

British Airways had no Content Security Policy. A single correctly configured header would have blocked the entire Magecart attack. In this challenge you analyse HTTP headers and identify what's wrong — and what the fix is.

AirNova HTTP response headers (before the attack)

HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8
Server: nginx/1.18.0
X-Powered-By: Express
Access-Control-Allow-Origin: *
Set-Cookie: session=eyJhbGc...; Path=/
X-Frame-Options: SAMEORIGIN

Red underlines = problems. Count them — there are 3 issues visible here, and one critical header is completely missing.

Q1. The Access-Control-Allow-Origin: * header means...

a) Only airnova.com can read responses from this server
b) Any website on the internet can make cross-origin requests and read the responses
c) The server blocks all cross-origin requests
d) SSL encryption is enabled
CORS wildcard (*) tells the browser to allow any origin to read responses — including attacker-controlled pages. If a user visits evil.com while logged into AirNova, that page can make authenticated requests to airnova.com and read the result. Fix: Access-Control-Allow-Origin: https://airnova.com

Q2. The Set-Cookie header is missing the HttpOnly flag. What does this allow an attacker to do?

a) Nothing — cookies are always safe
b) Read the session cookie via JavaScript: document.cookie
c) Delete the user's account
d) Bypass HTTPS
Without HttpOnly, any JavaScript on the page — including an injected XSS payload — can read document.cookie and steal the session token. Fix: Set-Cookie: session=...; HttpOnly; Secure; SameSite=Strict

Q3. Which header, if added, would have directly blocked the Magecart skimmer from sending stolen card data to an external server?

a) X-Frame-Options: DENY
b) Strict-Transport-Security: max-age=31536000
c) Content-Security-Policy: default-src 'self'; connect-src 'self'; img-src 'self' data:
d) X-Content-Type-Options: nosniff
CSP connect-src 'self' blocks fetch/XHR to external domains. img-src 'self' data: blocks the Image() exfiltration trick Magecart used. Together they would have prevented the stolen card data from ever leaving the browser — even with the skimmer injected. This single header was the £20 million fix.
FLAG{csp_w0uld_h4v3_st0pp3d_1t} — All 3 correct. You now understand the defence that British Airways didn't have. In your own projects: always set CSP, HttpOnly cookies, and a restricted CORS policy before going to production.
